# 📦 HealthHub: Dependency Architecture & Package Overview

This document provides a comprehensive list of the core technologies and libraries that power the HealthHub platform, categorized by service.

---

## 🖥 Backend (Node.js/Express)
The orchestration layer for data processing, security, and external integrations.

| Package | Purpose |
| :--- | :--- |
| **express** | High-performance web framework for building the REST API. |
| **mongoose** | ODM (Object Data Modeling) for MongoDB Atlas. |
| **jsonwebtoken** | Implements secure, stateless JWT-based authentication. |
| **bcryptjs** | Handles complex password hashing and verification. |
| **socket.io** | Enables real-time, bi-directional communication (Alerts, Triage updates). |
| **crypto (native)** | Used for AES-256 Symmetric Encryption of contact data. |
| **twilio** | Integration for sending emergency SMS alerts to donors. |
| **nodemailer** | Engine for dispatching clinical status and registration emails. |
| **multer** | Middleware for handling `multipart/form-data` during file uploads. |
| **dotenv** | Manages environment variables and secret keys across clinical environments. |
